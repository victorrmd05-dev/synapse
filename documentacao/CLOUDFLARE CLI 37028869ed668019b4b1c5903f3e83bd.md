# CLOUDFLARE CLI

**Wrangler** é a ferramenta oficial de CLI para desenvolver, compilar e fazer deploy de projetos do Cloudflare Workers e Pages.

Aqui está o que você pode fazer com ela:

- **Desenvolver localmente** — `wrangler dev` executa um servidor local de desenvolvimento com recarregamento automático
- **Fazer deploy** — `wrangler deploy` envia seu Worker para a rede de edge da Cloudflare
- **Gerenciar KV/R2/D1** — interaja com bindings como namespaces de KV, buckets do R2 e bancos de dados do D1
- **Acompanhar logs** — `wrangler tail` transmite logs em tempo real do seu Worker
- **Pages** — `wrangler pages dev` / `wrangler pages deploy` para projetos do Pages

**Início rápido:**

`npm create cloudflare@latest   # criar um novo projeto
wrangler dev                   # desenvolvimento local
wrangler deploy                # deploy em produção`

O que você gostaria de fazer com o Wrangler? Por exemplo:

- Criar um novo projeto de Worker
- Fazer deploy de um Worker existente
- Configurar bindings (KV, R2, D1 etc.)
- Solucionar um problema do Wrangler

INSTALAÇÃO 

Você quer configurar o **Claude Code** (da Anthropic) para que ele possa criar e publicar projetos no Cloudflare Workers e Pages. Isso requer gerar um **API Token** com as permissões corretas e configurá-lo no ambiente do Claude Code.

Vou buscar a documentação sobre isso:**Para dar acesso ao Claude Code ao seu Cloudflare e permitir criar/publicar Workers e Pages, siga estes passos:**

---

### **1. Instale o Claude Code (se ainda não tiver)**

`curl -fsSL https://claude.ai/install.sh | bash`

### **2. Instale o plugin Cloudflare no Claude Code**

Dentro do Claude Code, execute os dois comandos:

`/plugin marketplace add cloudflare/skills
/plugin install cloudflare@cloudflare`

Isso instala:

- **Cloudflare Skills** — conhecimento persistente sobre a plataforma (quando usar KV, D1, Durable Objects, etc.)
- **Cloudflare API MCP Server** — permite criar e gerenciar recursos via API (Workers, Pages, DNS, R2, etc.)
- **Wrangler Skill** — o Claude sabe quando rodar `npx wrangler deploy` e outros comandos CLI

### **3. Autorize o acesso à sua conta**

Na **primeira vez** que o Claude chamar uma ferramenta Cloudflare, você será redirecionado para uma **autenticação OAuth** no navegador. Escolha as permissões que deseja conceder.

### **4. Para CI/CD (sem OAuth interativo)**

Se quiser usar em ambientes sem interação (CI/CD), crie um **API Token** e passe como bearer token:

1. Acesse [Account API Tokens](https://dash.cloudflare.com/16da20022868f8dbfecb4e11fd57650c/api-tokens/create)
2. Crie um token com permissões de **Workers Scripts** (Edit) e **Pages** (Edit)
3. Configure no ambiente: `export CLOUDFLARE_API_TOKEN=seu_token_aqui`

### **5. Teste**

No Claude Code, tente um prompt como:

`Crie um Worker Hello World e publique na minha conta`

Ou, para Pages:

`Crie um site estático e publique como Pages project`

---

**Dica:** Execute o Claude Code a partir da raiz do seu projeto (onde está o `wrangler.jsonc`) para que ele entenda suas bindings e configurações automaticamente.
